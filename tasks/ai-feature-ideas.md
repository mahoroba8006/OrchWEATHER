# AIコメント機能 アイデア集

策定日: 2026-05-25  
ステータス: 検討中（未実装）

---

## 設計方針（共通）

- **ユーザー入力ゼロが絶対条件**：新たな入力・設定を求めない
- **天気情報画面に自然に溶け込む**：独立した設定画面ではなく、天気タブ内にコメントとして表示
- **AI エンジン**：Gemini 2.0 Flash（Firebase Functions 経由、APIキーはサーバーサイドのみ）
- **キャッシュ**：地点 + 日付 + 入力データハッシュをキーに Firestore にキャッシュ（TTL: 3〜6時間）
- **出力フォーマット**：プロンプトで箇条書き・短文を強制。長文・一般論は禁止指定

## アイデア 1：複合気象パターンの「翻訳」（隠れたリスクの言語化）

### 概要
単なる数値の羅列（気温〇度、湿度〇％）から、「それが農業的に何を意味するのか」をAIが言語化する。  
作物の種類が不明でも、普遍的な農業物理の視点からアドバイスが可能。

### AIへの入力データ
- 過去 3 日間の実績（気温・湿度・降水・日射）← Open-Meteo Historical API
- 向こう 3 日間の予報（同上）← Open-Meteo Forecast API
- 現在のリスクアラート（RiskSummary の計算結果）

### 出力例
> 「昨日までのまとまった雨で土壌が湿潤な中、明日は急激に気温が上がり強い日射が予想されます。  
> ハウス等の施設内では急激な"蒸れ"が発生しやすい気象パターンです。早めの換気を心がけてください。」

> 「明日の朝は気温が下がりますが、風が 4m/s 程度吹く予報のため、放射冷却による強烈な霜の降りるリスクは比較的低そうです。」

### ユーザーのメリット
人間が頭の中で行っている「複数の気象要素を掛け合わせた推測」をAIが代行。見落とし防止に直結。

### 配置タブ
天気タブ（メイン）

## アイデア 2：「作業の窓（チャンス）」の自動ピックアップ

### 概要
農業では「風が弱い時」「雨が降る前」「葉が乾いている時」など、特定条件が揃った時にしかできない作業がある。  
アプリ側で農作業適性条件を定義し、10日間の時間別予報からAIが最適タイミングを特定して提示する。

### AIへの入力データ
- 10 日間の時間別予報（風速・降水確率・気温・湿度）
- アプリ側で定義した作業適性条件（例：風速 2m/s 以下 かつ 前後 24 時間降水なし）

### 作業適性条件（アプリ側で定義する候補）
| 作業 | 条件 |
|------|------|
| 農薬散布 | 風速 < 2m/s、前後 24 時間降水なし、気温 10〜30℃ |
| ドローン飛行 | 風速 < 3m/s、降水なし |
| 天日干し | 終日快晴、湿度低、風あり |
| 圃場作業 | 雨上がり 24 時間以上経過、降水なし |

### 出力例
> 「🚜 向こう 1 週間で農薬散布に最も適しているのは【明後日（水）の午前 6〜9 時】です。  
> それを逃すと週末まで強風が続く予報です。」

> 「☀️ 明日は終日快晴で空気が乾燥します。収穫物の天日干しや農機のメンテナンスに最適な一日になりそうです。」

### ユーザーのメリット
「いつ作業しようか」と天気を読み解く手間がゼロになる。作業効率の直接的な向上。

### 配置タブ
天気タブ（メイン）

## アイデア 3：位置情報とカレンダーから「地域の旬」を推測する

### 概要
ユーザーに作物を入力させなくても、「今の時期・その場所で一般的に何が行われているか」をAIの一般知識から推測してアドバイスする。

### AIへの入力データ
- 緯度経度（+ 逆ジオコーディングで都道府県・地域名に変換してプロンプトに含める）
- 現在の月・時期（旬の推定用）
- 直近の気象実績・予報データ

### 出力例
> 「現在の気温推移は平年よりやや低めです。この地域では果樹の開花期にあたる時期ですが、  
> 明日の夜間は冷え込みが予想されるため、晩霜への警戒レベルを高めておくと安心です。」

> 「台風○号の接近に伴い、明後日から強風が予想されます。  
> この地域では収穫期を迎えている作物も多い時期です。飛散物の片付けや防風ネットの確認をおすすめします。」

### ユーザーのメリット
何も設定していないのに、地元のベテラン農家や JAからアドバイスを受けているような感覚。

### 技術メモ
- 逆ジオコーディング：Nominatim（無料）または Google Maps Geocoding API
- 地域名はキャッシュしてよい（地点は変わらないため）

### 配置タブ
天気タブ（メイン）

## アイデア 4：「去年との違い」の自動要約

### 概要
「今年の気候は昨年（平年）と比べてどうか」を、グラフを読む前にAIが一言で要約する。  
分析タブのグラフの「導入コメント」として表示するイメージ。

### AIへの入力データ
- 分析タブで選択中の「今年」と「昨年」の差分データ
  - 積算温度（GDD）の差（例：+10日分相当）
  - 累積降水量の差（例：昨年比 48%）
  - 気温トレンドの差（例：平均+2.3℃）

### 出力例
> 「📈 現在、今年の積算温度は昨年と比べて約 10 日分早く進行しています。  
> 春の訪れが早いパターンですので、昨年より早めの追肥・収穫準備を検討しても良いかもしれません。」

> 「💧 過去 1ヶ月の累積降水量が昨年同時期の半分以下です。極端な乾燥傾向。干ばつストレスに注意してください。」

### ユーザーのメリット
分析タブのグラフを読み解く労力ゼロ。一瞬で「今年の傾向」を把握できる。

### 配置タブ
分析タブ（グラフ上部のコメント欄として配置）

## 実装優先度（暫定）

| 優先 | アイデア | 理由 |
|------|---------|------|
| 高 | アイデア 2（作業ウィンドウ） | 即座に行動に結びつく。最も「便利」と感じてもらいやすい |
| 高 | アイデア 1（気象翻訳） | データは既存のものだけで完結。プロンプト設計のみ |
| 中 | アイデア 3（地域の旬） | 逆ジオコーディングの実装が追加で必要 |
| 中 | アイデア 4（去年との違い） | 分析タブの改修が別途必要 |

## 関連機能との接続

- **機能②（リスク閾値カスタマイズ）** → 実装後、アイデア 1・2 のプロンプトにユーザーの閾値を組み込むことで精度向上
- **機能①（過去気象再現）** → アイデア 4 の差分データ生成ロジックを共用できる可能性あり
- **機能③（地点登録拡充）** → アイデア 3 の逆ジオコーディングと実装を統合できる
