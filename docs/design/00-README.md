# Orch.Weather 設計書

本ディレクトリは、既存コードからのリバースエンジニアリングにより作成した設計書一式です。
**コード変更のたびに、該当する設計書も更新すること**を運用ルールとします。

## 記載ルール

- **コード値は「値＋意味」で明記する。** WMO天気コード、気象庁警報コード、`RiskType`、ティア区分などは、値の羅列ではなく各値の意味を併記する。
- **分岐・判断を網羅する。** モジュール設計書では、条件分岐・閾値・フォールバック・タイブレーク規則を漏れなく記載する（保守・障害解析の中核情報のため）。
- コード参照は `ファイル:行` 形式で示し、追跡可能にする。行番号は変更で陳腐化しうるため、関数名・記号名を併記する。
