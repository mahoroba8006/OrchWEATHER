# LP全面ブラッシュアップ 設計書（2026-06-12）

## 目的

ランディングページ（`src/components/LandingPage.tsx`）の内容・構成・デザインを刷新し、農業従事者に「使ってみたい」と思わせるLPにする。あわせてSafari等のクロスブラウザ互換性問題を解消する。

## 背景・現状診断

- 現状は13セクション・938行・ほぼ100%インラインstyle
- 課題①: 長すぎて主張が薄まる（SOLUTION・FEATURES・3連スポットライトで内容重複）
- 課題②: スクリーンショットが4箇所ともプレースホルダーのまま
- 課題③: 「現役農家が自分のために作った」という真正性ストーリーが活かされていない
- 課題④: NAVの `backdropFilter` に `-webkit-` prefix がない等、iOS Safari表示問題

## 決定事項

| 項目 | 決定 |
|---|---|
| 料金表示 | 当面「無料」のまま。料金を後で差し替え可能な構成にする |
| スクリーンショット | 今回追加する。開発サーバー＋ブラウザ操作で撮影（要ログイン協力）。困難な場合は実機スクショ提供にフォールバック |
| デザイン方向 | アプリ本体と統一（ティール `#0d9488` グラデーション＋ガラスモーフィズム「自然と水」） |
| コピー | 全面見直し可。ただしHero H1とPain 3引用は現行維持 |

## 新セクション構成（13 → 9）

```
1. NAV          ロゴ＋CTA（sticky・ガラス）
2. HERO         H1＋サブ＋CTA＋実機スクショ（CSSスマホフレーム）

4. MAKER'S NOTE 「作ったのは、同じ悩みを持つ農家です」（新設）
5. FEATURES     ジグザグ型ショーケース ×4行（スクショ付き）
6. COMPARISON   比較表 7行（Orch.Weather / 一般天気アプリ / 気象庁HP）

8. FINAL CTA    ティールグラデーション帯
```

- 旧3スポットライト＋6カードFEATURESはジグザグ4行に統合（「読むLP」→「見るLP」）
- 旧PRICING（¥0大型表示）は廃止し、HOW TO STARTのSTEP 1に「無料」を織り込む

## デザインシステム

| 要素 | 仕様 |
|---|---|
| アクセント | `#0d9488`、グラデーション `135deg, #0d9488 → #0f766e` |
| 背景 | アプリ本体と同じ淡いミント×水のグラデーション |
| カード | ガラス `rgba(255,255,255,0.75)` + `blur(12px)`、`-webkit-backdrop-filter` 併記 |
| フォント | Outfit + Inter（現行LPの `system-ui` 上書きを廃止） |
| 角丸 | `var(--radius-lg)` |
| CTA | ティールグラデーションのピルボタン（アプリのアクティブタブと同じ言語） |

## 実装方式

- 繰り返しスタイル（ガラスカード・CTA・セクション見出し等）→ 新規 `src/landing.css` のクラスに抽出。`-webkit-` prefix を一元管理
- 一回限りのレイアウト → インラインstyleのまま（過剰抽象化しない）
- `LandingPage.tsx` 1ファイル構成を維持、セクションごとに内部コンポーネント化

## Safari/クロスブラウザ対応

1. NAVの `backdropFilter` に `WebkitBackdropFilter` 併記（現在欠落）
2. `min-height: 100vh` → `100svh` フォールバック付き（`min-height: 100vh;` の後に `min-height: 100svh;`）
3. `position: sticky` に `-webkit-sticky` フォールバック
4. flex `gap`・`clamp()` は対応ブラウザ範囲内のため現状維持
