## 概要

現在モーダル形式の設定画面をタブ型UIに格上げし、ヘッダーを廃止してタブバーに統合する。  
同時に「現在地で登録」ボタンを地点設定に追加する。

## 1. 全体レイアウト変更

### Before

```

[タブバー: sticky 40px]      天気情報 │ 比較分析
```

### After

| アバター＋ログアウト | ヘッダー右端（常時表示） | Desktop: タブバー右端 / Mobile: 設定タブ内 |

- **右端（Desktop のみ）:** アバター画像（30px） ＋ ログアウトボタン

## 4. 現在地登録の動作フロー

### 正常フロー

```
[📍 現在地で登録] タップ
  → ボタンが「取得中…」（スピナー）に変化
  → ブラウザの位置情報許可ダイアログ
  → 許可
  → 緯度・経度取得
  → インライン編集フォームが開く
     地点名: 「現在地」（編集可能）
     緯度・経度: 自動入力（編集可能）
  → [保存] → Firestore 登録 → 一覧に追加
```

### エラーハンドリング

| ケース | 原因 | UIの挙動 |
|--------|------|---------|
| 許可拒否 | ユーザーがブラウザでブロック | ボタン直下に「⚠ 位置情報の許可が必要です。ブラウザの設定からオンにしてください」 |
| タイムアウト | GPS取得が10秒以内に未完了 | 「位置情報の取得に失敗しました。再試行してください」＋再試行ボタン |
| 非対応ブラウザ | `navigator.geolocation` が存在しない | ボタンを初期状態からグレーアウト・disabled |

### Geolocation オプション

```typescript
const GEO_OPTIONS: PositionOptions = {
  enableHighAccuracy: false,  // バッテリー消費を抑える（畑単位の精度で十分）
  timeout: 10000,             // 10秒
  maximumAge: 60000,          // 1分以内のキャッシュを再利用
};
```

### 地点名の初期値

`「現在地」`固定。ユーザーが自由に編集可能。  
（Reverse Geocoding は外部API依存のため今回スコープ外）

## 6. サブタブ③：比較分析

現在の `SettingsModal` から以下のセクションをそのまま移植する。

- **有効積算温度の基準温度**（基準温度1・2 ＋ 保存ボタン）
- **累積の開始日・日数差 表示設定**（4チャート分の開始日 ＋ 日数差閾値2つ ＋ 保存ボタン）

保存ロジック（`updateBaseTempSettings` / `updateAccumStartDates` / `updateAccumDeltaThresholds`）は既存のまま流用。
