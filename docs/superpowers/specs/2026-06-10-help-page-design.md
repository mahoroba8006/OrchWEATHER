# アプリの使い方ページ 設計書

## 概要

初めてのユーザーが各機能を迷わず使えるよう、アプリ内に「アプリの使い方」ページを追加する。
ヘッダーボタンから遷移し、4つのセクション（地点登録・いまの空・空くらべ・あの日の空）を縦スクロールで閲覧できる。

---

## ナビゲーション設計

### topTab 拡張

`topTab` の型を以下に拡張する：

```ts
'weather' | 'history' | 'analysis' | 'settings' | 'help'
```

### ヘッダーボタン配置

**デスクトップ**（`!isMobile`）
- 既存の「設定ギア + アバター + ログアウト」グループの左端に追加
- `HelpCircle` アイコン（size=18）＋「使い方」テキスト
- スタイルは既存の設定ギアボタンと同一（アクティブ時グリーングラデーション）

**モバイル**（`isMobile`）
- 既存の「アバター + 設定ギア」の左にアイコンのみボタンを追加
- `HelpCircle` アイコン（size=20）、テキストなし
- スタイルは設定ギアボタンと同一

### 戻る導線

HelpPage コンポーネントの最上部に「← 戻る」ボタンを置く。
App.tsx に `prevTopTab` を `useRef` で持ち、`'help'` に切り替える直前のタブを保存する。
「← 戻る」クリック時は `setTopTab(prevTopTab.current)` で遷移元タブに戻る。
初期値は `'weather'`。

---

## コンポーネント設計

### 新規ファイル

`src/components/HelpPage.tsx`

- Props: `onBack: () => void`
- 既存の `glass-panel` CSS クラスと `app-container` レイアウトを使用
- 外部ライブラリ追加なし

### App.tsx 変更点

1. `topTab` 型に `'help'` を追加
2. `prevTopTab` を `useRef<'weather' | 'history' | 'analysis' | 'settings'>('weather')` で追加
3. ヘルプボタンクリック時: `prevTopTab.current = topTab; setTopTab('help')`
4. デスクトップ・モバイル両ヘッダーにボタン追加
5. `{topTab === 'help' && <HelpPage onBack={() => setTopTab(prevTopTab.current)} />}` を追加

## 各セクションの内容

### 1. 地点登録の方法

3種類の登録方法を説明する。

| 方法 | 説明 |
|------|------|
| 現在地から取得 | ブラウザの位置情報を使って自動取得。毎回許可が必要な場合あり |
| マップから選択 | 地図上をタップ/クリックして任意の地点を登録 |
| 手動入力 | 地点名・緯度・経度を直接入力して登録 |

複数地点を登録しておくと「空くらべ」で比較できることも記載。

### 2. いまの空

**画面構成の説明**
- 時間別予報（2時間刻みのテーブル）
- 日別予報カード
- AIコメントカード（タブ切り替え）

**AIコメントの各タブ説明**

| タブ名 | 内容 |
|--------|------|
| 空ごよみ | 今日・明日の天気概況と数日先の傾向、作物への影響 |
| 畑しごと | 草取り・収穫・定植など外作業のタイミングと注意点 |
| 散布どき | 農薬・液肥の散布に適した条件と最適時間帯 |
| 施肥どき | 粒状・粉状肥料の施用タイミングの最適化提案 |
| 天気の備え | 荒天・乾燥・低温など気候リスクと備えの提案 |
| じぶん好み | 自分でプロンプトを入力して任意の回答を取得 |

**AIコメント設定**
- 設定タブで各タブの有効/無効を切り替え可能
- 無効にしたタブはAIコメントカードに表示されない

**じぶん好みタブ — AIに渡しているデータ一覧**

じぶん好みタブでは、以下のデータをAIに送信している。
プロンプトを書く際の参考にすること。

_時間別（1時間ごと × 48エントリ / 今後2日分）_

| フィールド | 内容 |
|------------|------|
| tmp | 気温（℃） |
| hum | 湿度（%） |
| ws | 風速（m/s） |
| wd | 風向（16方位） |
| wg | 瞬間風速（m/s） |
| pr | 降水量（mm） |
| pp | 降水確率（%） |
| snow | 降雪量（cm） |
| cape | CAPE（J/kg、対流不安定の指標） |
| frz | 0℃層高度（m） |
| prs | 海面気圧（hPa） |

_日別予報（3〜7日後 / 5日分）_ および _過去7日の日別実績_

| フィールド | 内容 |
|------------|------|
| tmpMax / tmpMin | 最高/最低気温（℃） |
| ppMax | 降水確率最大（%） |
| precip | 降水量（mm） |
| radSum | 日射量合計（MJ/m²） |
| sun | 日照時間（h） |
| wsMax | 最大風速（m/s） |

_その他_
- `now`：現在日時（日本時間）
- `location`：地点名
- `warnings`：気象庁の注意報・警報（発令中のもの）

**⚠️ ハルシネーション（AI の誤情報）に注意**

上記一覧にないデータをプロンプトで指定した場合、AIが**実際には存在しない数値を捏造して回答する**可能性があります。

例えば「今日の露点温度は？」「UV指数は？」「飽差は？」のように、渡していないデータを聞くと、AIはそれらしい数値を作り出してしまうことがあります。プロンプトは必ず上記の一覧にあるデータを根拠にした指示にしてください。

### 3. 空くらべ

**機能概要**
最大2地点・複数年のデータを並べて比較できる画面。年ごとの傾向把握や地点間の気象差の確認に使う。

**比較対象の設定方法**
- 地点と年を選択してターゲットを追加（最大2ターゲット）
- 気温・降水量・日射量・積算温度などをグラフで比較

**分析設定**
- 積算温度の基準温度（2種類設定可能）
- 各累計項目の集計開始日（降水量・日照・日射・積算温度）

**CSVダウンロード**
日別データをCSV形式でエクスポートできる。累計列・積算温度列も含まれる。

**⚠️ 2021年以前のデータについて**

2022年1月1日以降のデータは全項目取得できますが、**2021年12月31日以前**は以下の項目が取得できません：

- 降水確率
- UV指数
- 0℃層高度（9999固定）

該当期間のデータを見る際はご注意ください。

---

### 4. あの日の空

**機能概要**
任意の日付を指定して、その日の気象データ（時間別・日別）を参照できる画面。
過去の作業記録との照合や、特定イベント時の気象確認に使う。

**使い方**
- 地点を選択し、参照したい日付を入力すると気象データを取得・表示する

**⚠️ 取得できないデータについて**

「空くらべ」と同様、**2021年12月31日以前**のデータは以下の項目が取得できません：

- 降水確率
- UV指数
- 0℃層高度（9999固定）

## スタイル方針

- `app-container` + `glass-panel` の既存クラスを使用
- 新規CSSは最小限（セクション区切り線、目次リンクのアンカー設定のみ）
- アクセント色は既存の `var(--accent-color)`（teal）を踏襲
- テーブルは既存の `glass-table` クラスを使用
